# Threads

Threads let you reply to a specific message without cluttering the main channel conversation. They keep discussions focused and organized.

## Starting a Thread

Hover over any message and click **Reply in thread** (or the speech bubble icon). A thread panel opens on the right side of the screen, showing the original message and any replies.

## Replying in a Thread

Type your reply in the thread composer at the bottom of the thread panel. Thread replies are only visible inside the thread by default.

### Also Send to Channel

When replying in a thread, you can check **Also send to channel** to post the reply in both the thread and the main channel. This is useful for sharing a conclusion or important update with everyone.

## Thread Subscriptions

You are automatically subscribed to a thread when:

- You **post a reply** in it.
- Someone **replies to your message** (the parent message author is auto-subscribed on the first reply).

When subscribed, you receive notifications for new replies in the thread — even if the channel is muted.

You can **unsubscribe** from a thread to stop receiving notifications. Open the thread and use the thread menu to unsubscribe.

## Threads View

The **Threads** page (accessible from the sidebar or command palette) shows all threads you're subscribed to across the workspace. Each thread displays:

- The parent message
- Reply count and participants
- Last reply timestamp
- Unread indicator if there are new replies

## Thread Deep Links

Every thread has a shareable URL. Clicking a thread link opens the channel with the thread panel focused on that conversation.
